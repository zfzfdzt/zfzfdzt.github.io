hexo配置博客

发表于 2018-09-24 |

大家好，今天我想要写一个如何利用hexo搭建博客的教程。先说一下为什么要用hexo和github搭建静态网站，因为省钱。是的如果你用的是github自带的域名的话，那搭建这个博客甚至不用花一分钱，而且还可以把你的网站托管在gituhub上，可以说是非常的方便了。至于为什么要写博客，我觉得博客和写日记也没什么太大区别，如果你希望可以时不时回顾自己毫无亮点的人生，或者是记录自己表白失败的次数，那么拥有一个自己的博客听上去确实是一个不错的选择。另外我的电脑是Win10，一开始我是想用Jekyll，但是Jeky需要配置Ruby，这个和win10的兼容性大概是负数，所以我选择利用hexo来搭建静态博客。

环境配置

  • 下载安装Node.Js，点链接进去选择自己电脑的配置，无脑往后就可以了，安装时默认会安装npm的。安装好后运行cmd输入node -v，如果安装成功会显示当前版本。
  • 下载安装git，进入官网自己选择下载安装。安装好后运行cmd输入git version，如果安装成功会显示当前版本。
  • 下载安装hexo，打开cmd，输入$ npm install hexo-cli -g，我之前直接安装的时候貌似不行，应该是需要科学上网之后才能正确安装。另外如果是mac系统则需要输入$ sudo npm install hexo-cli -g。

还有一些话

如果你选择用hexo搭建博客，那么就意味着你要用markdown这种轻量级标记语言来进行编辑。相比于大家都用的word或者wps之类，这代表着所见不是所得。你在编辑器内写的符号可能不会出现在生成的文本中，但是他们可以帮助你有效控制你的文本美观度。无论是插入代码还是插入图片，它都会自动帮你排版好，你只需要认真思考你的博客需要写什么就可以了。事实上，markdown作为一个轻量级的语言，从入门到使用大概只要半个小时，大家不要有什么心理压力。网上关于markdown的教程多如牛毛，我这里推荐毫无编程基础的人先学习一下markdown的基础配置，比如如何写标题，如何生成预览pdf，剩下的一些小问题诸如怎么把字放大或者怎么插入图片可以在写作时谷歌百度，基本是用一次就知道了。如果大家想要做一个排版优良的微信公众号，那markdown应该也是必修的科目了。
markdown只适用于一些简单的文本编辑，涉及到报告之类的复杂文章，我还是推荐大家用latex。这里有几个我朋友写的小教程，看完入门没啥太大问题。虽然说如果word用的好的可以做到和latex编译的没什么区别，但是如果word用的这么炉火纯青，花费的精力就不是一般的多了。至于编译时出现的一些小问题，大家学会用谷歌和百度就不会有什么太多解决不了的事。安利的话就说这么多，下次再见咯！
